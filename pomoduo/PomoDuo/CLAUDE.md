## Project Overview

PomoDuo is an iOS application built using Tuist for project management and modular architecture. The project is in early development stages.

## Build System

This project uses **Tuist** for Xcode project generation and dependency management. Tuist is managed via `mise` (see `mise.toml`).

### Essential Commands

```bash
# Install/use the correct Tuist version
mise install

# Generate Xcode project files (must be run after any Project.swift changes)
mise exec -- tuist generate

# Clean generated files
mise exec -- tuist clean

# Install/update dependencies
mise exec -- tuist install

# Build the project
mise exec -- tuist build

# Run tests
mise exec -- tuist test

# Run tests for a specific module
mise exec -- tuist test -p Modules/PomoDuoHTTPService
```

**Important**: The `.xcodeproj` and `.xcworkspace` files are git-ignored and generated by Tuist. Never edit them directly - modify `Project.swift` files instead and regenerate.

## Architecture

### Modular Structure

The project follows a modular architecture with separate targets:

- **Main App** (`PomoDuo/`): The iOS application target
  - Sources: `PomoDuo/Sources/`
  - Resources: `PomoDuo/Resources/`
  - Tests: `PomoDuo/Tests/`

- **Modules** (`Modules/`): Reusable framework modules
  - `PomoDuoHTTPService`: HTTP networking layer (macOS framework for now)
    - Each module has its own `Project.swift` defining targets and dependencies

### Project Configuration

Project structure is defined in Tuist manifests:

- **`Project.swift`**: Main project configuration defining app targets
- **`Modules/*/Project.swift`**: Individual module configurations
- **`Tuist/Package.swift`**: External dependencies (Swift Package Manager)
- **`Tuist.swift`**: Tuist configuration settings
- **`mise.toml`**: Tool version management (Tuist 4.102.0)

### Dependency Injection

The project uses the **Factory** dependency injection library (configured in `Tuist/Package.swift`). Services are registered as Factory instances in Container extensions and can be injected using the `@Injected` property wrapper or accessed directly via `Container.shared`.

### HTTP Service Layer (`PomoDuoHTTPService` Module)

The networking module provides `PomoDuoHTTPService`, a generic, type-safe HTTP client built on URLSession with maximum flexibility - URLs are specified per request rather than per instance.

**Key Features**:
- **GET requests**: Generic `get<Response>(from url: URL, responseType: Response.Type, headers: [String: String]?)` supporting:
  - Type-safe response with `Decodable` types
  - Custom headers via optional parameter
  - Automatic JSON decoding
  - Error handling for HTTP errors, decoding failures via `PomoDuoHTTPServiceError`
- **POST requests**: Generic `post<Request, Response>(to url: URL, ...)` supporting:
  - Type-safe request/response with `Codable` types
  - Custom headers via optional parameter
  - Automatic JSON encoding/decoding
  - Error handling for HTTP errors, encoding/decoding failures via `PomoDuoHTTPServiceError`

**Example usage**:
```swift
import FactoryKit

struct Task: Codable {
    let id: String
    let title: String
    let duration: Int
}

struct CreateTaskRequest: Codable {
    let title: String
    let duration: Int
}

struct CreateTaskResponse: Codable {
    let id: String
    let created: Bool
}

// Using dependency injection (recommended)
class TaskService {
    @Injected(\.pomoDuoHTTPService) private var httpService

    func fetchTask(id: String) async throws -> Task {
        try await httpService.get(
            from: URL(string: "https://api.example.com/tasks/\(id)")!,
            responseType: Task.self,
            headers: ["Authorization": "Bearer token"]
        )
    }

    func createTask(title: String, duration: Int) async throws -> CreateTaskResponse {
        try await httpService.post(
            to: URL(string: "https://api.example.com/tasks")!,
            CreateTaskRequest(title: title, duration: duration),
            responseType: CreateTaskResponse.self,
            headers: ["Authorization": "Bearer token"]
        )
    }
}

// Or access directly via Container
let httpService = Container.shared.pomoDuoHTTPService()
let task = try await httpService.get(
    from: URL(string: "https://api.example.com/tasks/123")!,
    responseType: Task.self,
    headers: ["Authorization": "Bearer token"]
)
```

**Design**: URL-per-request pattern allows different modules to use the same HTTP service instance for different endpoints, avoiding hardcoded URLs and improving testability. The module is specifically focused on HTTP API calls.

**Testing**: Uses URLProtocol-based mocking (see `Tests/Mocks/MockURLProtocol.swift`). Tests demonstrate header injection, error handling, and body encoding verification.

## Project Conventions

### File Organization

- Use `buildableFolders` pattern in main app: specify source/resource directories
- Use `sources` pattern in modules: use glob patterns for file discovery
- Keep module source files in `Sources/`, tests in `Tests/`

### Adding Dependencies

1. Add external dependencies to `Tuist/Package.swift`
2. Reference them in target dependencies with `.external(name: "PackageName")`
3. Configure `productTypes` in `PackageSettings` if needed (e.g., static vs dynamic framework)
4. Run `mise exec -- tuist install` to fetch dependencies
5. Run `mise exec -- tuist generate` to update Xcode projects

### Creating New Modules

1. Create directory under `Modules/YourModule/`
2. Add `Project.swift` defining the module target
3. Organize code in `Sources/` and tests in `Tests/`
4. Reference module from main app with `.target(name: "YourModule")` dependency
5. Generate the specific module project: `mise exec -- tuist generate -p Modules/YourModule`
6. To include in workspace, regenerate from root: `mise exec -- tuist generate`

## Development Workflow

1. **Before starting**: Run `mise exec -- tuist generate` to ensure Xcode projects are current
2. **After changing Project.swift files**: Always regenerate with `mise exec -- tuist generate`
3. **After pulling changes**: Run `mise exec -- tuist install && mise exec -- tuist generate`
4. **Open project**: Use `PomoDuo.xcworkspace`, not the `.xcodeproj`
